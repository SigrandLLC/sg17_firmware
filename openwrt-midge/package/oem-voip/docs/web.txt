	Описание настройки VoIP-модулей, производимое через web-интерфейс.

--------------
	После загрузки web-интерфейса во вкладке "system information" можно
ознакомиться с наличием VoIP-модулей и с каналами на них (их типами и
номерами). Для настройки VoIP следует открыть вкладку "Hardware->VoIP" в меню
слева. Здесь будут представлены несколько вкладок, предоставляющих доступ к
VoIP-настройкам. Далее дано описание содержимого вкладок.

--------------
1. <Settings>
--------------
	Так как спецификация на протокол RTP не указывает явно на то, какой номер
порта следует использовать для пересылки траффика, поля "RTP port start" и
"RTP port end" предоставляют возможность задать диапазон значений, в
котором программа, устанавливающая соединение будет пытаться открыть порт для 
пересылки RTP-траффика.
	"Logging level" - поле, в котором можно задать степень подробности
отчётов, создаваемых программой в процессе своей работы. Можно задать от "off"
- отсутствие каких-либо отчётов, до "9" - наиболее подробные отчёты. "0,1,2" -
уровни, на которых в логи пишется информация об ошибках. С повышением уровня
логгирования добавляется информация о установлении SIP-соединений и даже о
порядке вызовов функций программы.
	В "SIP settings" можно задать координаты sip-сервера, имя учётной записи и
пароль. При наведении курсора мыши на поля возникает подсказка с форматом
вводимых значений. Можно, скажем, зарегистрироваться на сервере sipnet.ru,
зайдя на сайт, ознакомившись с правилами и условиями регистрации и т.д.
Полученный там логин и пароль ввести здесь. Теперь, после запуска, программа
будет пытаться соединиться с сервером и получить доступ к указанному аккаунту.
Поле "FXS channel" позволяет установить номер канала, на который будут
переадрисовываться звонки, приходящие от пользователей internet на ваш
sip-аккаунт.

--------------
2. <Hotline>
--------------
	Здесь можно настроить режим "горячей линии". Если есть hotline-соединение
от FXS канала к другому FXS (FXS_1->FXS_2), то при подъёме трубки на FXS_1
происходит автоматический дозвон до FXS_2. На стороне FXS_2 телефон начинает
звонить, чтобы ответить на звонок, надо просто поднять трубку. 
	При типе соединения FXS->FXO после подъёма трубки на FXS стороне,
происходит дозвон до FXO-окончания. Когда в трубке будут слышен сигнал,
выдаваемый АТС, нужно донабрать номер и ожидать соединения. Таким образом
можно получить соединение FXS -- FXO -- PSTN -- Remote phone. 
	При типе соединения FXO->FXS звонок с АТС, подключённой к FXO-порту
перенаправляется на указанный FXS порт, реализуя схему 
Remote phone -- PSTN -- FXO -- FXS.
	Чтобы включить hotline-режим для определённого канала, нужно в строке с
номером этого канала поставить галочку в поле hotline, а в поле "Complete
number" указать адрес удалённой точки. Адрес указывается в формате 
<r_id>-<ch_id>, где r_id - идентификационный номер роутера (см. описание для
вкладки "Routes"), на котором находится точка, а ch_id - идентификационный номер канала 
удалённой точки. Этот адрес может выглядеть, например так: "101-12" или 
так "*-3". В первом случае удалённая точка находится на роутере с номером 101 
на канале 12, во втором случае удалённая точка находится на собственном рутере 
(* раскрывается как номер собственного роутера) на 3-м канале.

--------------
3. <Hardlink>
--------------
	Здесь можно настроить 2-х и 4-х -проводные ТЧ-соединения. Для 2-х
проводных соединений (2w) используется один канал. Для 4-х проводного (4w)
используется FXS-модуль целиком. 1-й канал на модуле работает в режиме
передатчика (TX), 2-й канал - в режиме приёмника (RX). Если удалённая точка
ТЧ-соединения находится на удалённом рутере, то на нём также необходимо
добавить настройку ТЧ-соединения с соответствующими параметрами. Если
удалённая точка находится на том же роутере, то необходимо прописать
соединение лишь в "одном нарпавлении", то есть не нужно как раньше делать две
записи для обоих направлений.
Пример записи:
	("4w", "00/01", "*", "02/03", "aLaw", "20", "receiver"),
"*" в поле Router ID распознаётся как собственный номер роутера. Последнее
поле определяет кто будет устанавливать соединение. Если ТЧ канал разнесён на
два роутера, то "reciver"-роутер должен быть включён перед "caller"-роутеом,
чтобы тот мог до него дозвониться.
	Mожно также выбирать используемй кодак, и устанавить для него
время пакетизации, payload и тип упаковки битов. Если нужно изменить уровень 
усиления сигнала и прочие параметры, относящиеся к rtp-протоколу, то это можно 
сделать во вкладке для установки rtp-параметров в строках для соответствующих 
каналов. 

--------------
4. <Routes>
--------------
	В этой вкладке хранится таблица с роутерами сети. Здесь нужно перечислить
все роутеры, если их больше одного, дав им уникальные номера и указав ip-адреса
для доступа к ним. Указанные здесь номера роутеров используются в других
вкладках, где требуется указать адрес доступа.

--------------
5. <Addresses>
--------------
	Это адресная книга. Здесь можно задать короткий номер ("short number") и
подразумеваемый под этим номером номер дозвона ("complete number"). Короткий
номер - это двухзначное положительное число. Номер дозвона должен содержать
<route_id>-<chan_id>. Для обозначения собственного роутера можно также
использовать "*". Когда задана пара "short number" -> "complete number", То
при снятии трубки на любом FXS канале на роутере (кроме "hotline" и
"hardlink" каналов) и набре "#<short number>" происходит дозвон по номеру,
указанному в "complete number".

--------------
6. <RTP>
--------------
	На этой вкладке можно настроить способ передачи DTMF-сигналов для каналов,
громкость, детектор активности в канале, и высокочастотный фильтр.
	"OOB" - настройка способа передачи DTMF-сигналов. 
		"in-band" - передавать наравне с остальным RTP-траффиком (имеет смысл
				при использовании g711 кодака).
		"out-of-band" - кодировать в специальные RTP-кадры (см. rfc2833).
		"both" - использовать оба способа.
		"block" - не передавать DTMF-сигналы.
	"OOB_play" - что делать при получении out-of-band DTMF-кадра.
		"play" - декодировать и проигрывать.
		"mute" - заглушать.
		"play_diff_pt" - то же, что и "play", но используется в случае, если
				выбран нестандартный маркер для out-of-band кадров.
	"nEventPT" и "nEventPlayPT" - значение маркера для out-of-band кадра
			передачи DTMF при передаче и приёме соответственно.
	"Tx_vol" и "Rx_vol" - настройка громкости в канале на передаче/приёме.
	"VAD" - Voice Activity Detector - выбор режима работы для детектора
			активности в канале.
		"off"- выключить всё (VAD, SC, CN) - Voice activity detector, silence
				compression, comfort noice.
		"on" - включить VAD для генерация комфортного шума (comfort noice) с 
				использованием информации о спектре (выс. качество).
		"g711 - включить VAD для генерация комфортного шума без использования
				информации о спектре (низ. качество).
		"CNG_only" - включить VAD для генерации комфортного шума, но без 
				сжатия "пустых" кадров (silence compression).
		"SC_only" - включить VAD только для сжатия "пустых" кадров.
	"HPF" - High-Pass Filter - фильтр высоких частот. Отсекает артефакты при
			кодировании/декодировании речи. "on" / "off" - включить /
			выключить соответственно.
	Стандартные настройки в этой секции работают весьма сносно. Изменяйте
что-либо на свой страх и риск.
		
--------------
7. <Quality>
--------------
	В этой вкладке можно установить приоритет использования определённых
кодаков и их параметры. Здесь три секции. "Internal" - определяет набор
кодаков, их приоритеты и параметры для связи внутри локальной сети из
маршрутизаторов. "External" - аналогично, но для связи через internet
(соединения, устанавливаемые через sip-сервер в сети internet). "Fax" - можно
выбрать кодак, используемый для передачи факса по протоколу T.38.
В "Internal" и "External" можно упорядочить используемые кодаки по
предпочтению. В первую очередь будут использоваться кодаки с наименьшим
номером "Priority". Для каждого кодака можно выбрать время пакетизации
("paketization time") и идентификатор типа кодака в RTP-потоке ("payload").
Для кодаков серии g726 можно также определить способ упаковки кадра
("bitpack"). Если вы не знаете что всё это такое и зачем нужно - оставьте 
как есть.

--------------
8. <WLEC>
--------------
	WLEC - Window-based Line Echo Canceller. 
Здесь можно выставить параметры эхоподавления. для каждого канала в
отдельности. Значения всех параметров прописаны в веб-интерфейсе.
